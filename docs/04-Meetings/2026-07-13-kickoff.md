## Key Decisions

1. **Separate Memory** — HiveOS gets its own dedicated knowledge base (this `docs/` directory), independent from Hossein's main Obsidian vault (`D:\mind\mine`).
   - Location: `~/Desktop/hive-os/`
   - Opens as Obsidian vault (`.obsidian/` config included)
   - Version-controlled with Git
   - Structured with numbered prefixes

2. **Hermes as Runtime** — Every agent in a flow runs as a Hermes session/delegate_task. We don't build a new agent framework; we orchestrate Hermes instances.

3. **Flow DSL First** — The first concrete deliverable is a declarative YAML schema for defining multi-agent workflows.

4. **Package Format** — standard tar.gz bundle of flows + skills + knowledge + config, installable on any Hermes instance via `hermes package install`.

## Conversation Log

### What Hossein Wants

- A **Multi-Agent OS** where you can define any flow and have multiple agents work together as a full team
- Each agent has its own **skills** and **knowledge**
- A **playground** to experiment with different agent configurations
- Everything should be a **package** — set up Hermes on another system, give it a package, and it immediately works like the original
- A **Mothership** agent on one system — other agents on other systems get their knowledge and skills from the Mothership
- Agents execute tasks within their assigned flow

### Architecture Concepts Discussed

1. **Workflow DSL** — YAML-defined agent teams with roles, dependencies, triggers
2. **Flow Engine** — reads DSL, spawns agents via delegate_task, chains outputs
3. **Agent Registry** — Mothership tracks all nodes, their capabilities, and assigned flows
4. **Package Format** — portable ecosystem bundles
5. **Communication Bus** — cross-node agent messaging
6. **Resilience** — node failure handling, task reassignment

### Memory Strategy

- Dedicated knowledge base outside main vault
- Every conversation = a meeting note with `#` tags
- Architecture Decision Records (ADRs) for key decisions
- Live ROADMAP.md updated after every session
- Hermes memory (2KB) used for quick-access pointers only
