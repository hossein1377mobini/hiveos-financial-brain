# مبحث ۱: اصول حسابداری — مفاهیم پایه

**تاریخ:** 2026-07-13
**وضعیت:** ✅ کامل
**منبع:** ترکیبی از منابع آموزشی حسابداری ایران

## ۱. تعریف حسابداری

**حسابداری (Accounting)** فرآیند شناسایی، اندازه‌گیری، ثبت، طبقه‌بندی، تلخیص و گزارش‌گری رویدادهای مالی یک واحد تجاری است. هدف نهایی آن، فراهم کردن اطلاعات معنی‌دار برای تصمیم‌گیری مدیران، سرمایه‌گذاران، بانک‌ها و سایر ذی‌نفعان است.

> "حسابداری زبان تجارت است" — Accounting is the language of business

---

## ۲. عناصر پایه حسابداری

| عنصر | انگلیسی | تعریف |
|------|---------|-------|
| **دارایی** | Assets | منابع اقتصادی تحت کنترل واحد تجاری که از رویدادهای گذشته به دست آمده و انتظار می‌رود منافع آتی داشته باشد |
| **بدهی** | Liabilities | تعهدات فعلی واحد تجاری که از رویدادهای گذشته ناشی شده و باید در آینده تسویه شود |
| **سرمایه (حقوق صاحبان سهام)** | Equity | باقیمانده دارایی‌ها پس از کسر بدهی‌ها = حق مالکانه صاحبان واحد تجاری |
| **درآمد** | Revenue / Income | افزایش در حقوق صاحبان سهام ناشی از ارائه کالا یا خدمات |
| **هزینه** | Expenses | کاهش در حقوق صاحبان سهام ناشی از فعالیت‌های واحد تجاری |

### معادله اصلی حسابداری

```
دارایی‌ها = بدهی‌ها + سرمایه
Assets = Liabilities + Equity
```

این معادله **پایه تمام سیستم حسابداری دوطرفه (Double-Entry)** است. هر تراکنش مالی این معادله را متعادل نگه می‌دارد.

### مثال عددی

شرکت الف: ۵۰۰ میلیون تومان دارایی (نقد+کالا+ماشین‌آلات)، ۲۰۰ میلیون بدهی (وام+بستانکار)، ۳۰۰ میلیون سرمایه

```
۵۰۰ = ۲۰۰ + ۳۰۰ ✅
```

## ۳. اصول و مفروضات بنیادی حسابداری

### مفروضات (Assumptions)

| فرض | توضیح |
|-----|-------|
| **تفکیک شخصیت** (Economic Entity) | حساب‌های شخصی مالک از حساب‌های کسب‌وکار جدا است |
| **تداوم فعالیت** (Going Concern) | واحد تجاری برای آینده قابل پیش‌بینی به فعالیت ادامه می‌دهد |
| **دوره مالی** (Time Period) | عمر واحد تجاری به دوره‌های زمانی مشخص (معمولاً یک سال) تقسیم می‌شود |
| **واحد پولی** (Monetary Unit) | فقط رویدادهای قابل اندازه‌گیری با پول ثبت می‌شوند |

### اصول (Principles)

| اصل | توضیح | مثال |
|-----|-------|------|
| **ثبت به بهای تمام شده** (Historical Cost) | دارایی‌ها به قیمت خرید ثبت می‌شوند، نه ارزش بازار | ساختمانی به قیمت خرید ۱ میلیارد ثبت می‌شود، حتی اگر الان ۳ میلیارد ارزش داشته باشد |
| **تحقق درآمد** (Revenue Recognition) | درآمد زمانی شناسایی می‌شود که کالا/خدمت ارائه شده، نه لزوماً هنگام دریافت وجه | فروش نسیه ۱۰۰ میلیون → درآمد شناسایی می‌شود، حتی اگر پول هنوز وصول نشده |
| **تطابق** (Matching) | هزینه‌ها در همان دوره‌ای شناسایی می‌شوند که درآمد مرتبط شناسایی می‌شود | هزینه حقوق فروشندگان در دوره فروش ثبت می‌شود |
| **افشای کامل** (Full Disclosure) | تمام اطلاعات بااهمیت باید در صورت‌های مالی افشا شود | بدهی‌های احتمالی در یادداشت‌های توضیحی ذکر شود |
| **ثبات رویه** (Consistency) | روش‌های حسابداری در دوره‌های مختلف یکسان باشد | اگر استهلاک به روش خط مستقیم محاسبه می‌شود، در همه دوره‌ها همینطور باشد |
| **اهمیت** (Materiality) | فقط موارد بااهمیت ثبت شوند | خرید ۵۰ هزار تومان خودکار را نمی‌توان دارایی محسوب کرد (بلکه هزینه است) |
| **محافظه‌کاری** (Conservatism / Prudence) | در عدم اطمینان، روشی انتخاب شود که دارایی‌ها و درآمدها را کمتر و بدهی‌ها و هزینه‌ها را بیشتر نشان دهد | زیان احتمالی شناسایی می‌شود اما سود احتمالی شناسایی نمی‌شود |

## ۴. ثبت حسابداری دوطرفه (Double-Entry Accounting)

### قانون طلایی

| نوع حساب | افزایش | کاهش |
|----------|--------|------|
| **دارایی** (Assets) | **بدهکار** (Debit) | بستانکار (Credit) |
| **هزینه** (Expenses) | **بدهکار** | بستانکار |
| **بدهی** (Liabilities) | بستانکار | **بدهکار** |
| **سرمایه** (Equity) | بستانکار | **بدهکار** |
| **درآمد** (Revenue) | بستانکار | **بدهکار** |

### فرمول ثبت

```
نام حساب بدهکار       مبلغ
    نام حساب بستانکار     مبلغ
شرح: توضیح تراکنش
```

### مثال‌های عملی

**۱. پرداخت نقدی اجاره ۱۰ میلیون تومان:**

```
هزینه اجاره          ۱۰,۰۰۰,۰۰۰
    صندوق (نقد)                 ۱۰,۰۰۰,۰۰۰
شرح: پرداخت اجاره محل شرکت
```

**۲. خرید نسیه کالا به مبلغ ۵۰ میلیون:**

```
خرید کالا            ۵۰,۰۰۰,۰۰۰
    بستانکاران (حساب‌های پرداختنی)    ۵۰,۰۰۰,۰۰۰
شرح: خرید کالا از شرکت تامین
```

**۳. دریافت وجه نقد از بدهکار (بستانکاران قدیمی):**

```
صندوق               ۲۰,۰۰۰,۰۰۰
    بدهکاران (حساب‌های دریافتنی)      ۲۰,۰۰۰,۰۰۰
شرح: وصول وجه از مشتریان
```

**۴. پرداخت حقوق کارکنان ۱۵ میلیون:**

```
هزینه حقوق          ۱۵,۰۰۰,۰۰۰
    صندوق                          ۱۵,۰۰۰,۰۰۰
شرح: پرداخت حقوق ماهانه
```

## ۵. صورت‌های مالی پایه

### ترازنامه (Balance Sheet / Statement of Financial Position)

وضعیت مالی در یک لحظه خاص را نشان می‌دهد:

```
شرکت نمونه
ترازنامه
تاریخ: ۳۱ اسفند ۱۴۰۴

دارایی‌ها:
  دارایی‌های جاری:
    نقد و بانک                    ۸۰,۰۰۰,۰۰۰
    حساب‌های دریافتنی            ۱۲۰,۰۰۰,۰۰۰
    موجودی کالا                  ۱۵۰,۰۰۰,۰۰۰
  دارایی‌های ثابت:
    ساختمان                      ۵۰۰,۰۰۰,۰۰۰
    ماشین‌آلات                   ۲۰۰,۰۰۰,۰۰۰
    (-) استهلاک انباشته          (۵۰,۰۰۰,۰۰۰)
─────────────────────────────────────
جمع دارایی‌ها                   ۱,۰۰۰,۰۰۰,۰۰۰

بدهی‌ها و سرمایه:
  بدهی‌های جاری:
    حساب‌های پرداختنی            ۱۰۰,۰۰۰,۰۰۰
    وام کوتاه مدت                  ۵۰,۰۰۰,۰۰۰
  بدهی‌های بلندمدت:
    وام بلندمدت                  ۱۵۰,۰۰۰,۰۰۰
  سرمایه:
    سرمایه صاحب شرکت            ۵۰۰,۰۰۰,۰۰۰
    سود انباشته                  ۲۰۰,۰۰۰,۰۰۰
─────────────────────────────────────
جمع بدهی‌ها و سرمایه           ۱,۰۰۰,۰۰۰,۰۰۰
```

### صورت سود و زیان (Income Statement)

عملکرد شرکت در یک دوره را نشان می‌دهد:

```
شرکت نمونه
صورت سود و زیان
برای سال منتهی به ۳۱ اسفند ۱۴۰۴

فروش                                    ۸۰۰,۰۰۰,۰۰۰
(-) بهای تمام شده کالای فروش رفته      (۴۵۰,۰۰۰,۰۰۰)
───────────────────────────────────────
سود ناخالص                              ۳۵۰,۰۰۰,۰۰۰
(-) هزینه‌های عملیاتی:
   حقوق و دستمزد                         ۱۲۰,۰۰۰,۰۰۰
   هزینه اجاره                            ۴۰,۰۰۰,۰۰۰
   هزینه استهلاک                          ۳۰,۰۰۰,۰۰۰
   سایر هزینه‌ها                           ۵۰,۰۰۰,۰۰۰
───────────────────────────────────────
سود خالص قبل از مالیات                  ۱۱۰,۰۰۰,۰۰۰
(-) مالیات بر درآمد                      (۲۵,۰۰۰,۰۰۰)
───────────────────────────────────────
سود خالص پس از مالیات                     ۸۵,۰۰۰,۰۰۰
```

## ۶. نکات کلیدی برای ایران

1. **استانداردهای حسابداری ایران** توسط سازمان حسابرسی تدوین می‌شود و عمدتاً برگرفته از **IFRS** است اما با برخی تعدیلات بومی
2. **سال مالی** در ایران معمولاً از **۱ فروردین تا ۲۹ اسفند** است (اما برخی شرکت‌ها سال مالی متفاوت دارند)
3. **صورت‌های مالی** باید به ریال تهیه شود و واحد پول گزارشگری ریال ایران است
4. **سامانه مودیان** از ۱۴۰۲ الزامی شده — ثبت صورتحساب الکترونیکی در سامانه مودیان برای همه مشمولین
5. **نرم‌افزارهای حسابداری** رایج در ایران: همکاران سیستم، سپیدار، هلو، پارسیان، راهکاران
