## تعریف حسابداری

حسابداری (Accounting) یک سیستم اطلاعاتی است که با شناسایی (Identification)، ثبت (Recording)، طبقه‌بندی (Classification)، تلخیص (Summarization) و گزارش‌گری (Reporting) رویدادهای مالی، اطلاعات لازم را برای تصمیم‌گیری در اختیار استفاده‌کنندگان قرار می‌دهد.

حسابداری را «زبان کسب‌وکار» (Language of Business) می‌نامند، زیرا اطلاعات مالی را به صورتی استاندارد و قابل فهم برای همگان ارائه می‌کند.

**هدف اصلی حسابداری:** ارائه اطلاعات مفید برای تصمیم‌گیری‌های اقتصادی و پاسخگویی (Accountability).

### استفاده‌کنندگان اطلاعات حسابداری

| دسته | مصادیق |
|------|--------|
| **درون سازمانی (Internal)** | مدیران، هیئت مدیره، کارکنان |
| **برون سازمانی (External)** | سرمایه‌گذاران، اعتباردهندگان، بانک‌ها، سازمان امور مالیاتی، تأمین اجتماعی، بورس اوراق بهادار |

## چارچوب نظری گزارشگری مالی ایران

چارچوب نظری گزارشگری مالی ایران (Iranian Conceptual Framework for Financial Reporting) توسط **سازمان حسابرسی (Audit Organization of Iran)** تدوین شده است. این چارچوب مبنای تدوین استانداردهای حسابداری ایران را تشکیل می‌دهد و به عنوان «پیوست استانداردهای حسابداری - مفاهیم نظری گزارشگری مالی» منتشر شده است.

**مرجع تدوین‌کننده:** سازمان حسابرسی (Audit Organization of Iran) — تأسیس ۱۳۶۶ بر اساس قانون تشکیل سازمان حسابرسی مصوب ۱۳۶۲/۱۱/۴

**اساس قانونی:** بند ۴ تبصره ۲ قانون تشکیل سازمان حسابرسی و مواد ۶ و ۷ اساسنامه سازمان حسابرسی (مصوب ۱۳۶۶/۶/۲۶)

**نهاد مسئول تدوین استانداردها:** کمیته تدوین استانداردهای حسابداری (Accounting Standards Committee) — تأسیس ۱۳۷۱

### ساختار چارچوب نظری

1. **هدف صورت‌های مالی** — ارائه اطلاعات درباره وضعیت مالی، عملکرد مالی و انعطاف‌پذیری مالی
2. **استفاده‌کنندگان** — سرمایه‌گذاران، اعتباردهندگان، و سایر ذی‌نفعان
3. **خصوصیات کیفی اطلاعات مالی**
4. **مفروضات بنیادی**
5. **اصول حسابداری**
6. **عناصر صورت‌های مالی** (دارایی‌ها، بدهی‌ها، حقوق صاحبان سرمایه، درآمدها، هزینه‌ها)

### همگرایی با IFRS

- ایران اعلام کرده است که الزامات IFRS را برای شرکت‌های پذیرفته شده در بورس، بانک‌ها، بیمه‌ها و مؤسسات مالی الزامی کرده است.
- سازمان حسابرسی به عنوان مرجع تدوین استانداردهای ملی، برنامه همگرایی (Convergence) با IFRS را دنبال می‌کند.
- استانداردهای حسابداری ایران عمدتاً برگرفته از IAS/IFRS هستند اما با شرایط بومی ایران تطبیق داده شده‌اند.

## مفروضات بنیادی حسابداری

مفروضات حسابداری (Accounting Assumptions) فرض‌های اولیه و بنیادینی هستند که زیربنای اصول و استانداردهای حسابداری را تشکیل می‌دهند. این مفروضات در چارچوب نظری گزارشگری مالی ایران تصریح شده‌اند.

### ۱. فرض تفکیک شخصیت (Separate Entity Assumption / Entity Concept)

واحد تجاری (Business Entity) یک شخصیت مستقل و مجزا از مالکان و سایر واحدهای اقتصادی تلقی می‌شود. رویدادهای مالی واحد تجاری باید جدا از رویدادهای مالی شخصی مالکان ثبت و گزارش شود.

**مثال کاربردی:** اگر مالک شرکت برای خرید شخصی از حساب شرکت برداشت کند، این مبلغ به عنوان **برداشت (Drawings / برداشت صاحب سرمایه)** ثبت می‌شود، نه هزینه شرکت.

### ۲. فرض تداوم فعالیت (Going Concern Assumption)

فرض بر این است که واحد تجاری برای آینده قابل پیش‌بینی به فعالیت خود ادامه خواهد داد و قصد انحلال یا توقف فعالیت ندارد. این فرض مبنای استفاده از بهای تمام‌شده تاریخی (Historical Cost) را توجیه می‌کند.

**استثنا:** اگر شواهدی مبنی بر ورشکستگی یا انحلال وجود داشته باشد، باید از مبنای **ارزش تصفیه (Liquidation Value)** استفاده شود.

### ۳. فرض دوره مالی (Periodicity Assumption / Time Period Assumption)

عمر نامحدود واحد تجاری به دوره‌های زمانی مشخص (معمولاً یک ساله) تقسیم می‌شود تا عملکرد مالی در هر دوره قابل اندازه‌گیری باشد. سال مالی در ایران معمولاً از **۱ فروردین تا ۲۹ اسفند** (مطابق سال شمسی) است، اما شرکت‌ها می‌توانند سال مالی متفاوتی (مثلاً ۱ مهر تا ۳۱ شهریور) انتخاب کنند.

### ۴. فرض واحد پولی (Monetary Unit Assumption)

رویدادهای مالی باید بر حسب واحد پول رایج کشور اندازه‌گیری و ثبت شوند. در ایران، واحد پول **ریال (Rial)** است (هرچند در عمل تومان نیز کاربرد دارد). این فرض ایجاب می‌کند که فقط رویدادهایی که قابل اندازه‌گیری پولی هستند در حسابداری ثبت شوند.

**محدودیت:** اثر تورم در این فرض نادیده گرفته می‌شود — یعنی قدرت خرید پول در طول زمان تغییر می‌کند اما در حسابداری لحاظ نمی‌گردد.

### ۵. فرض تعهدی (Accrual Basis Assumption)

رویدادهای مالی در زمان وقوع شناسایی می‌شوند، نه در زمان دریافت یا پرداخت وجه نقد. این فرض مبنای **حسابداری تعهدی (Accrual Accounting)** است.

## اصول حسابداری

اصول حسابداری (Accounting Principles) قواعد و دستورالعمل‌های اجرایی هستند که نحوه ثبت، اندازه‌گیری و گزارش رویدادهای مالی را مشخص می‌کنند.

### ۱. اصل بهای تمام‌شده تاریخی (Historical Cost Principle)

دارایی‌ها و بدهی‌ها باید در زمان تحصیل یا ایجاد به بهای تمام‌شده (Historical Cost) ثبت شوند. بهای تمام‌شده معادل ارزش مبادله نقدی در تاریخ تحصیل است.

**مثال:** شرکت زمینی به قیمت ۵,۰۰۰,۰۰۰,۰۰۰ ریال خریداری می‌کند. دو سال بعد ارزش بازار زمین به ۸,۰۰۰,۰۰۰,۰۰۰ ریال افزایش می‌یابد. طبق این اصل، زمین همچنان به قیمت ۵,۰۰۰,۰۰۰,۰۰۰ ریال در دفاتر ثبت می‌ماند (مگر در موارد کاهش ارزش که استاندارد ۳۲ الزام به تجدید ارزیابی داشته باشد).

> **مزایا:** قابلیت تأیید (Verifiability)، عینی بودن (Objectivity)
> **معایب:** عدم انعکاس ارزش جاری در شرایط تورمی — این ضعف در ایران با استفاده از **تجدید ارزیابی دارایی‌ها (Revaluation)** در موارد مجاز تا حدی جبران می‌شود.

### ۲. اصل تحقق درآمد (Revenue Recognition Principle)

درآمد زمانی شناسایی می‌شود که:
1. **قابل اندازه‌گیری** (Measurable) باشد
2. **مربوط** (Relevant) باشد
3. **قابل اتکا** (Reliable) باشد
4. فرآیند کسب درآمد **کامل** شده باشد
5. مزایا و خطرات مالکیت **انتقال** یافته باشد

**روش‌های شناسایی درآمد:**

| روش | توضیح | کاربرد |
|-----|-------|--------|
| روش فروش (Sales Basis) | در لحظه فروش شناسایی می‌شود | رایج‌ترین روش |
| روش وصول (Collection Basis) | پس از وصول وجه نقد شناسایی می‌شود | فروش نسیه با وصول مشکوک |
| روش اقساطی (Installment Method) | به نسبت اقساط وصولی شناسایی می‌شود | فروش اقساطی |
| روش درصد تکمیل (Percentage of Completion) | بر اساس پیشرفت کار شناسایی می‌شود | پیمان‌های بلندمدت |
| روش تکمیل کار (Completed Contract) | پس از اتمام کامل پروژه شناسایی می‌شود | پیمان‌های کوتاه‌مدت |

### ۳. اصل تطابق هزینه با درآمد (Matching Principle)

هزینه‌ها باید در همان دوره‌ای شناسایی شوند که درآمد مرتبط با آنها شناسایی می‌شود. این اصل رابطه علت و معلولی بین هزینه‌ها و درآمدها را تأکید می‌کند.

**سه نوع رابطه:**

| نوع | توضیح | مثال |
|-----|-------|------|
| **ارتباط مستقیم** (Direct Association) | هزینه مستقیماً با درآمد خاصی مرتبط است | بهای تمام‌شده کالای فروش رفته |
| **تخصیص سیستماتیک** (Systematic Allocation) | هزینه به دوره‌های منتفع تخصیص می‌یابد | استهلاک دارایی ثابت |
| **هزینه دوره** (Period Cost) | هزینه در دوره وقوع شناسایی می‌شود | حقوق مدیر، اجاره |

### ۴. اصل افشای کامل (Full Disclosure Principle)

کلیه اطلاعات بااهمیت (Material) که می‌تواند بر تصمیم‌گیری استفاده‌کنندگان تأثیر بگذارد، باید به طور کامل در صورت‌های مالی یا یادداشت‌های توضیحی افشا شود.

**روش‌های افشا:**
- در متن صورت‌های مالی
- در یادداشت‌های توضیحی (Notes to Financial Statements)
- در گزارش هیئت مدیره
- در گزارش حسابرس مستقل

### ۵. اصل ثبات رویه (Consistency Principle)

واحد تجاری باید از روش‌های حسابداری یکسان در دوره‌های متوالی استفاده کند، مگر اینکه تغییر رویه به بهبود ارائه اطلاعات مالی منجر شود. در صورت تغییر رویه، باید دلایل و اثرات آن در یادداشت‌های توضیحی افشا شود.

### ۶. اصل اهمیت (Materiality Principle)

فقط رویدادهایی بااهمیت محسوب می‌شوند که اندازه یا ماهیت آنها بتواند بر تصمیم‌گیری استفاده‌کنندگان تأثیر بگذارد. رویدادهای کم‌اهمیت ممکن است با روش‌های ساده‌تری ثبت شوند.

### ۷. اصل محافظه‌کاری (Conservatism / Prudence Principle)

در شرایط عدم اطمینان، حسابدار باید جانب احتیاط را رعایت کند:
- شناسایی هزینه‌ها و بدهی‌های احتمالی **به محض امکان**
- شناسایی درآمدها و دارایی‌های احتمالی **فقط در زمان تحقق**

## خصوصیات کیفی اطلاعات حسابداری

خصوصیات کیفی (Qualitative Characteristics) ویژگی‌هایی هستند که اطلاعات مالی را برای تصمیم‌گیری مفید می‌سازند. بر اساس «مفاهیم نظری گزارشگری مالی ایران» (پیوست استانداردهای حسابداری):

### خصوصیات کیفی اصلی (Fundamental Qualitative Characteristics)

#### ۱. مربوط بودن (Relevance)
اطلاعات زمانی مربوط است که بتواند در تصمیم‌گیری‌های اقتصادی کاربران تأثیر بگذارد.

**مؤلفه‌ها:**
- **ارزش پیش‌بینی‌کنندگی (Predictive Value):** کمک به پیش‌بینی نتایج آینده
- **ارزش تأییدکنندگی (Confirmatory Value):** تأیید یا تصحیح پیش‌بینی‌های قبلی
- **اهمیت (Materiality):** حذف یا اشتباه در آن می‌تواند بر تصمیم تأثیر بگذارد

#### ۲. قابل اتکا بودن (Reliability)
اطلاعات زمانی قابل اتکا هستند که عاری از اشتباه با اهمیت، بی‌طرفانه و بیانگر صادقانه (Faithful Representation) واقعیت‌های اقتصادی باشند.

**مؤلفه‌ها:**
- **بیان صادقانه (Faithful Representation):** انعکاس صحیح ماهیت اقتصادی رویداد
- **بی‌طرفی (Neutrality):** عدم سوگیری در ارائه اطلاعات
- **کامل بودن (Completeness):** ارائه تمام اطلاعات لازم
- **احتیاط (Prudence):** رعایت جانب احتیاط در شرایط عدم اطمینان
- **رجحان محتوا بر شکل (Substance over Form):** ثبت رویداد بر اساس ماهیت اقتصادی، نه شکل حقوقی

### خصوصیات کیفی ارتقادهنده (Enhancing Qualitative Characteristics)

| ویژگی | توضیح |
|-------|-------|
| **قابل مقایسه بودن (Comparability)** | امکان مقایسه اطلاعات یک واحد در دوره‌های مختلف و بین واحدهای مختلف |
| **قابل فهم بودن (Understandability)** | اطلاعات باید برای استفاده‌کنندگان با آگاهی معقول قابل درک باشد |
| **به‌موقع بودن (Timeliness)** | اطلاعات باید قبل از از دست دادن تأثیرگذاری بر تصمیمات ارائه شود |
| **قابل تأیید بودن (Verifiability)** | ناظران مستقل بتوانند با روش‌های مشابه به نتایج یکسان برسند |

### محدودیت‌های خصوصیات کیفی

- **محدودیت هزینه - منفعت (Cost-Benefit Constraint):** منافع حاصل از اطلاعات باید از هزینه تهیه آن بیشتر باشد

| صنعت خاص | برخی صنایع نیازمند رویه‌های خاص هستند |

## معادله بنیادی حسابداری

معادله حسابداری (Accounting Equation) اساس سیستم حسابداری دوطرفه (Double-Entry Accounting) است:

```
دارایی‌ها (Assets) = بدهی‌ها (Liabilities) + حقوق صاحبان سرمایه (Equity)
```

### عناصر معادله

| عنصر | تعریف | مثال‌ها |
|------|-------|---------|
| **دارایی (Asset)** | منابع اقتصادی تحت کنترل واحد تجاری که از رویدادهای گذشته ناشی شده و انتظار می‌رود منافع اقتصادی آتی را به جریان اندازد | وجه نقد، حساب‌های دریافتنی، موجودی کالا، زمین، ساختمان، ماشین‌آلات، سرمایه‌گذاری‌ها، دارایی نامشهود (حق اختراع، نرم‌افزار) |
| **بدهی (Liability)** | تعهد فعلی واحد تجاری ناشی از رویدادهای گذشته که تسویه آن منجر به خروج منابع خواهد شد | حساب‌های پرداختنی، اسناد پرداختنی، وام بانکی، پیش‌دریافت از مشتریان، ذخایر |
| **حقوق صاحبان سرمایه (Equity)** | باقیمانده دارایی‌ها پس از کسر بدهی‌ها | سرمایه، اندوخته‌ها، سود (زیان) انباشته |

### مثال محاسباتی

فرض کنید شرکت «الف» دارایی‌های زیر را دارد:
- وجه نقد: ۲,۰۰۰,۰۰۰,۰۰۰ ریال
- حساب‌های دریافتنی: ۳,۰۰۰,۰۰۰,۰۰۰ ریال
- موجودی کالا: ۵,۰۰۰,۰۰۰,۰۰۰ ریال
- دارایی ثابت: ۱۰,۰۰۰,۰۰۰,۰۰۰ ریال

بدهی‌ها:
- حساب‌های پرداختنی: ۴,۰۰۰,۰۰۰,۰۰۰ ریال
- وام بانکی: ۶,۰۰۰,۰۰۰,۰۰۰ ریال

**محاسبه حقوق صاحبان سرمایه:**
```
کل دارایی‌ها = ۲ + ۳ + ۵ + ۱۰ = ۲۰ میلیارد ریال
کل بدهی‌ها = ۴ + ۶ = ۱۰ میلیارد ریال
حقوق صاحبان سرمایه = ۲۰ - ۱۰ = ۱۰ میلیارد ریال
```

## مبنای حسابداری تعهدی

حسابداری تعهدی (Accrual Accounting) در مقابل حسابداری نقدی (Cash Accounting) قرار دارد. در ایران، استانداردهای حسابداری بر مبنای **تعهدی کامل (Full Accrual Basis)** است.

### مقایسه مبنای نقدی و تعهدی

| رویداد | مبنای نقدی | مبنای تعهدی |
|--------|-----------|------------|
| فروش کالا به صورت نسیه | ثبت نمی‌شود تا وجه واصل شود | در لحظه فروش به عنوان درآمد ثبت می‌شود |
| خرید مواد اولیه نسیه | ثبت نمی‌شود تا وجه پرداخت شود | به عنوان خرید و بدهی ثبت می‌شود |
| پیش‌دریافت از مشتری | به عنوان درآمد ثبت می‌شود | به عنوان بدهی (پیش‌دریافت) ثبت می‌شود |
| حقوق پرداختنی پایان دوره | ثبت نمی‌شود | به عنوان هزینه و بدهی (حقوق پرداختنی) ثبت می‌شود |

### انواع اقلام تعهدی

1. **هزینه‌های پرداختنی (Accrued Expenses):** هزینه‌هایی که در دوره جاری شناسایی شده اما تا پایان دوره پرداخت نشده‌اند (مثال: حقوق پایان دوره)
2. **درآمدهای دریافتنی (Accrued Revenues):** درآمدهایی که در دوره جاری شناسایی شده اما تا پایان دوره وصول نشده‌اند (مثال: سود سرمایه‌گذاری)
3. **پیش‌پرداخت (Prepaid Expenses):** هزینه‌هایی که پرداخت شده اما به دوره‌های آینده تعلق دارند (مثال: بیمه پیش‌پرداخت)
4. **پیش‌دریافت (Unearned Revenues):** وجوهی که دریافت شده اما درآمد آن به دوره‌های آینده تعلق دارد (مثال: پیش‌دریافت اجاره)

## مثال‌های عملی

### مثال ۱: ثبت رویدادها بر اساس اصول حسابداری

شرکت «ایران‌نوین» در تاریخ ۱۴۰۴/۱۲/۱۵ کالایی به ارزش ۵۰۰,۰۰۰,۰۰۰ ریال به صورت نسیه (۳۰ روزه) فروخته است. بهای تمام‌شده کالا ۳۰۰,۰۰۰,۰۰۰ ریال است.

**ثبت حسابداری بر اساس اصل تحقق درآمد و اصل بهای تمام‌شده:**

| تاریخ | شرح | بدهکار (ریال) | بستانکار (ریال) |
|-------|-----|---------------|-----------------|
| ۱۴۰۴/۱۲/۱۵ | حساب‌های دریافتنی | ۵۰۰,۰۰۰,۰۰۰ | |
| | فروش | | ۵۰۰,۰۰۰,۰۰۰ |
| | *(ثبت فروش نسیه — اصل تحقق درآمد)* | | |
| ۱۴۰۴/۱۲/۱۵ | بهای تمام‌شده کالای فروش رفته | ۳۰۰,۰۰۰,۰۰۰ | |
| | موجودی کالا | | ۳۰۰,۰۰۰,۰۰۰ |
| | *(ثبت هزینه مرتبط — اصل تطابق)* | | |

### مثال ۲: ثبت هزینه پرداختنی (مبنای تعهدی)

شرکت «ایران‌نوین» حقوق و دستمزد اسفند ۱۴۰۴ به مبلغ ۲۰۰,۰۰۰,۰۰۰ ریال را در فروردین ۱۴۰۵ پرداخت کرده است. ثبت در پایان سال ۱۴۰۴:

| تاریخ | شرح | بدهکار (ریال) | بستانکار (ریال) |
|-------|-----|---------------|-----------------|
| ۱۴۰۴/۱۲/۲۹ | حقوق و دستمزد | ۲۰۰,۰۰۰,۰۰۰ | |
| | حقوق پرداختنی | | ۲۰۰,۰۰۰,۰۰۰ |
| | *(ثبت هزینه تعهدی — اصل تطابق)* | | |

### مثال ۳: استهلاک دارایی ثابت (تخصیص سیستماتیک)

شرکت دستگاهی به قیمت ۱,۲۰۰,۰۰۰,۰۰۰ ریال با عمر مفید ۵ سال و ارزش اسقاط ۲۰۰,۰۰۰,۰۰۰ ریال خریداری کرده است.

**محاسبه استهلاک به روش خط مستقیم (Straight-Line):**
```
مبلغ استهلاک سالانه = (بهای تمام‌شده - ارزش اسقاط) / عمر مفید
مبلغ استهلاک سالانه = (۱,۲۰۰,۰۰۰,۰۰۰ - ۲۰۰,۰۰۰,۰۰۰) / ۵
مبلغ استهلاک سالانه = ۲۰۰,۰۰۰,۰۰۰ ریال
```

| تاریخ | شرح | بدهکار (ریال) | بستانکار (ریال) |
|-------|-----|---------------|-----------------|
| سالانه | هزینه استهلاک | ۲۰۰,۰۰۰,۰۰۰ | |
| | استهلاک انباشته | | ۲۰۰,۰۰۰,۰۰۰ |
| | *(ثبت استهلاک — اصل تطابق و تخصیص سیستماتیک)* | | |

### مثال ۴: اصل افشای کامل

شرکت «ایران‌نوین» در دادخواستی به مبلغ ۱,۰۰۰,۰۰۰,۰۰۰ ریال به عنوان متهم معرفی شده است. وکلای شرکت احتمال محکومیت را **محتمل** (Probable) و مبلغ را **قابل برآورد** تخمین می‌زنند.

**طبق استاندارد ۴ (ذخایر، بدهی‌های احتمالی و دارایی‌های احتمالی):**

| تاریخ | شرح | بدهکار (ریال) | بستانکار (ریال) |
|-------|-----|---------------|-----------------|
| ۱۴۰۴/۱۲/۲۹ | هزینه حقوقی (ذخیره دعوی) | ۱,۰۰۰,۰۰۰,۰۰۰ | |
| | ذخیره دعوی حقوقی | | ۱,۰۰۰,۰۰۰,۰۰۰ |
| | *(اصل محافظه‌کاری و افشای کامل)* | | |

> **نکته:** اگر احتمال محکومیت **ممکن** (Possible) باشد، ذخیره ثبت نمی‌شود اما در یادداشت‌های توضیحی افشا می‌گردد. اگر احتمال محکومیت **دور** (Remote) باشد، نیازی به ثبت یا افشا نیست.
